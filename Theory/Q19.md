# Q19. (Theory) What is the difference between analogWrite() and analogRead() in Arduino? What is PWM and why is it used? Give a practical IoT example for each. (2 marks)
## Key Differences
### *analogRead():* Reads a continuous input voltage (0V to 5V) from the real world. It uses an internal converter to translate that voltage into a digital number between 0 and 1023.
### *analogWrite():* Simulates a variable voltage output. It does this by rapidly blinking a digital pin on and off using PWM, using values from 0 to 255.
### **PWM** (*Pulse Width Modulation*) is a trick where a digital pin switches between 5V (ON) and 0V (OFF) thousands of times per second. By changing the ratio of "ON time" to "OFF time" (called the duty cycle), it tricks electronic components into behaving as if they are receiving a steady intermediate voltage (like 2.5V instead of 5V). It is used because microcontrollers cannot output true, varying analog voltages on their own.
## Practical IoT Examples
### *analogRead()* Example: A Smart Farm Soil Sensor checking continuous moisture levels to decide if crops need water.
### *analogWrite()* (PWM) Example: A Smart Thermostat Fan that spins faster or slower depending on how hot a server room gets.
