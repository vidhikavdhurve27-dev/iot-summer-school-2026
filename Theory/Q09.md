# Q9. (Theory) Explain the difference between git pull, git fetch, and git clone. When would you use each? (2 marks)

## Key Differences 
### *git clone:* Downloads a complete, brand-new copy of a remote repository onto your computer for the first time. 
### *git fetch:* Downloads the latest history and changes from GitHub without modifying your active local code.
### *git pull:* Downloads remote changes and instantly merges them into your current local branch (git fetch + git merge).

## When to Use Each 
### *git clone:* When starting a project or downloading a peer's repository for the first time.
### *git fetch:* When checking what your team changed without risking breaking your current workspace.
### *git pull:* When updating your local branch with the latest code from GitHub before you start writing new code.
