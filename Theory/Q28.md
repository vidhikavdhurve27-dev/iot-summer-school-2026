# Q29. (Theory) What is sensor calibration and why is it important in IoT systems? How would you calibrate an analog soil moisture sensor to give accurate percentage readings? Describe the two-point calibration method. (2 marks)
## Definition and Importance
### Sensor calibration is the process of mapping raw electronic output values from a sensor (like voltages or ADC counts) against known, accurate physical standards. It is critical in IoT systems because mass-produced sensors suffer from manufacturing variances, component aging, and environmental interference. Without calibration, an IoT device will log inaccurate data, leading to faulty automated decisions (such as overwatering crops due to skewed moisture telemetry).
## Calibrating an Analog Soil Moisture Sensor
### An analog soil moisture sensor outputs raw analog integers from 0 to 1023. To convert these arbitrary values into an accurate \(0\%\) to \(100\%\) scale, you must apply the Two-Point Calibration Method:
### Point 1 (Dry Reference / 0%): Hold the sensor completely dry in the air. Record the highest stable raw value from the Serial Monitor (e.g., 850). This represents \(0\%\) moisture.
### Point 2 (Wet Reference / 100%): Submerge the sensor probes completely in a cup of water. Record the lowest stable raw value (e.g., 290). This represents \(100\%\) moisture.
### Code Mapping- Use the Arduino map() function to convert raw runtime sensor inputs into a linear percentage based on these two boundaries:
int rawValue = analogRead(A0);
// Reverse the constraints because a lower voltage means higher moisture
int moisturePercentage = map(rawValue, 850, 290, 0, 100); 
moisturePercentage = constrain(moisturePercentage, 0, 100); // Guard rails
