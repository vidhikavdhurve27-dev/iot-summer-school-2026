# Q20. (Theory) Explain the setup() and loop() functions in Arduino. What happens if you put a long delay() inside loop()? How does this affect sensor reading responsiveness? What is the non-blocking alternative? (2 marks)
## *setup():* Runs exactly once when the Arduino turns on or resets. It is used to initialize pins, start serial communication, or configure sensors.
## *loop():* Runs continuously over and over in an infinite cycle for as long as the Arduino has power. This is where your main program logic lives.
## The Danger of a Long delay()
### When you use a command like delay(5000), the microcontroller completely freezes all execution for 5 seconds.
## Impact on Responsiveness: 
### The Arduino cannot check pins, process incoming data, or read sensors during this window. If a user presses a button or an emergency sensor triggers an alert during those 5 seconds, the system will completely miss it.
## **The Non-Blocking Alternative**
### The professional alternative is to use the millis() function. Instead of forcing the CPU to sit idle, millis() acts like a stopwatch that counts the number of milliseconds passed since the program started. Your code checks this stopwatch on every single loop cycle to see if it is time to perform an action. This keeps the processor completely free to read sensors and handle user inputs instantly.
